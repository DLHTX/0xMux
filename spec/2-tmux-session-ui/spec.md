# 功能规格说明：Tmux 会话管理全功能 UI

**版本**: 1.0
**日期**: 2026-02-09
**状态**: 草案
**分支**: `2-tmux-session-ui`

---

## 概述

为 0xMux 实现核心的 tmux 会话管理界面。用户可以在浏览器中以类终端的暗色极客风格 UI 完成所有 tmux 会话的增删改查操作，并通过内嵌终端（xterm.js + WebSocket PTY）直接在浏览器中与 tmux 会话交互。界面支持分屏、多窗口 Tab、拖拽调整大小，同时适配移动端设备。

---

## 目标用户

- **主要用户**: 使用 tmux 管理多个终端会话的开发者，需要在浏览器中集中管理和操作 session
- **次要用户**: 在手机/平板上远程查看和操作 tmux session 的开发者

---

## 用户场景

### 场景 1：首次进入 — 空状态体验

**角色**: 开发者
**前置条件**: 0xMux 服务已启动，系统中没有任何 tmux session
**流程**:

1. 用户打开 0xMux 界面
2. 顶部 Header 显示 0xMux 像素风格 Logo，右侧显示设置图标
3. 主区域显示终端命令行风格的空状态页面，带有闪烁光标效果
4. 页面中央有一个醒目的「+ 新建 Session」按钮
5. 用户点击按钮，弹出创建对话框，输入 session 名称（可选默认名称）
6. 创建成功后，界面自动切换到主布局（左侧列表 + 右侧终端）

**预期结果**: 空状态页引导用户完成第一个 session 的创建，过渡到主界面自然流畅

### 场景 2：日常会话管理 — 列表操作

**角色**: 开发者
**前置条件**: 系统中存在多个 tmux session
**流程**:

1. 用户打开 0xMux，左侧面板显示所有 session 列表
2. 每个 session 卡片显示：名称、状态指示灯（attached=绿色/detached=灰色）、窗口数量
3. 用户点击某个 session 名称，右侧终端区域打开该 session 的终端视图
4. 用户双击 session 名称进入行内编辑模式，输入新名称后按 Enter 确认重命名
5. 用户点击 session 卡片上的删除图标，弹出确认对话框，确认后删除 session
6. 用户在顶部搜索框输入关键词，列表实时过滤匹配的 session
7. 用户点击「+ 新建」按钮创建新的 session

**预期结果**: 所有 CRUD 操作均可在列表面板中完成，操作结果实时反映在 UI 上

### 场景 3：终端交互 — 分屏与多窗口

**角色**: 开发者
**前置条件**: 已选中一个 tmux session
**流程**:

1. 右侧工作区显示选中 session 的终端视图，渲染真实的终端输出（颜色、光标等）
2. 用户在终端中输入命令并看到实时输出
3. 用户 hover 窗格左上角显示操作按钮，点击分屏按钮（或使用快捷键），将当前终端区域水平/垂直分为两个窗格
4. 每个窗格连接到不同的 tmux session（同一 session 不允许在多个窗格中同时打开）
5. 用户从侧边栏拖拽 session 到窗格中心区域可替换该窗格的 session，拖到边缘则分屏
6. 用户拖拽窗格之间的分隔线调整大小
7. session 的多个 window 以 Tab 形式显示在终端区域顶部，点击切换
8. 用户可以创建新 window Tab 或关闭已有 Tab
9. 终端区域自动适应窗口大小变化（resize 事件通知后端调整 PTY 大小）

**预期结果**: 用户获得类似本地终端复用器的体验，支持多 session 多 window 并行操作

### 场景 4：移动端使用

**角色**: 在手机上远程操作的开发者
**前置条件**: 手机浏览器通过同一网络访问 0xMux
**流程**:

1. 手机端打开 0xMux，显示全屏的 session 列表页
2. 用户点击某个 session，全屏切换到终端视图
3. 用户通过顶部返回按钮或左滑手势回到列表
4. 长按 session 卡片弹出操作菜单（重命名、删除）
5. 虚拟键盘弹出时，终端视图自动缩小适应可见区域
6. 底部导航栏提供快速切换 session 列表和当前终端的入口

**预期结果**: 移动端用户可以完成所有基本的 session 管理和简单终端操作，布局适配小屏幕

### 场景 5：设置与偏好

**角色**: 开发者
**前置条件**: 0xMux 正在运行
**流程**:

1. 用户点击 Header 右侧设置图标
2. 弹出设置面板，包含：
   - 终端字体大小调整
   - 主题颜色偏好（默认霓虹绿，可自定义强调色）
   - 默认分屏布局偏好
   - WebSocket 连接状态显示
3. 设置变更即时生效，保存到浏览器本地存储

**预期结果**: 用户可以个性化终端显示偏好

## 功能需求

### FR-1: Header 与 Logo 展示

- 顶部固定 Header 栏，高度约 48px
- 左侧显示 0xMux 像素风格/艺术字 Logo，点击回到首页
- 右侧显示功能图标组：连接状态指示、设置按钮
- 图标使用 Iconify 图标库
- Header 在所有页面状态下保持一致

### FR-2: 空状态页面

- 无 session 时显示终端命令行风格的空状态页
- 包含闪烁光标动画效果，模拟等待用户输入
- 中央显示醒目的「创建 Session」操作按钮
- 可选：显示引导提示文字（如 `> 创建你的第一个 tmux 会话...`）

### FR-3: Session 列表面板

- 左侧固定宽度面板（桌面端），显示所有 tmux session
- 每个 session 条目包含：名称、状态指示灯、窗口数量
- 支持创建新 session（点击「+」按钮弹出命名对话框）
- 支持删除 session（带确认对话框，防止误删）
- 支持重命名 session（双击名称进入行内编辑，Enter 确认，Esc 取消）
- 列表顶部搜索框，实时过滤 session
- 面板支持折叠/展开（点击边缘的折叠按钮或拖拽）
- 选中的 session 高亮显示
- 实时更新：当后端检测到 tmux session 变化时，列表自动刷新

### FR-4: 终端视图与 xterm.js 集成

- 使用 xterm.js 在浏览器中渲染真实终端
- 通过 WebSocket 连接后端 PTY 进程
- 支持完整的终端功能：文本输入/输出、ANSI 颜色、光标定位、终端铃声
- 终端大小自适应容器，resize 时通知后端调整 PTY 窗口大小
- 支持 URL 链接点击（xterm-addon-web-links）
- 终端背景透明/半透明，融入整体暗色主题
- 终端获得焦点时边框高亮

### FR-5: 分屏功能

- 支持将终端工作区水平或垂直分为多个窗格
- 每个窗格独立运行一个终端实例，可连接不同的 session
- 窗格之间的分隔线可通过拖拽调整比例
- 最小窗格尺寸限制（防止窗格被拖拽到不可见）
- 支持关闭单个窗格（剩余窗格自动扩展填满空间）
- 分屏操作入口：每个窗格左上角悬浮按钮（hover 显示） + 右键菜单
- 拖拽 session 到窗格边缘触发分屏，拖到中心区域替换当前窗格的 session
- 同一个 session 不能同时在多个窗格中打开（防止重复连接）

### FR-6: 多窗口 Tab 功能

- 每个 session 内的多个 window 以 Tab 形式展示
- Tab 栏位于终端区域顶部，显示 window 索引和名称
- 支持点击切换 Tab、新建 Tab、关闭 Tab
- 当前活动 Tab 高亮
- Tab 数量超出可视区域时支持横向滚动

### FR-7: 移动端适配

- 断点 768px 以下进入移动端布局
- 移动端：session 列表为全屏页面，隐藏左侧面板
- 点击 session 全屏切换到终端视图
- 提供返回按钮和滑动手势从终端返回列表
- 长按 session 触发操作菜单（重命名、删除）
- 虚拟键盘弹出时自动调整终端可视区域
- 底部导航栏：快速切换「列表」和「终端」

### FR-8: 设置面板

- 点击 Header 设置图标打开设置面板（侧滑或弹窗）
- 可配置项：终端字体大小（12-24px）、强调色选择、默认分屏方向
- 设置保存到浏览器 localStorage
- 变更即时生效，无需刷新

### FR-9: 后端 PTY WebSocket 端点

- 后端新增 WebSocket 端点用于 PTY 连接
- 每个终端实例对应一个独立的 WebSocket 连接
- 支持终端输入/输出的双向传输
- 支持终端 resize 消息
- 连接断开时自动重连（客户端指数退避策略）
- 后端使用 PTY 库创建和管理伪终端进程

### FR-10: 实时状态同步

- Session 列表数据通过现有 WebSocket 实时推送更新
- 新增/删除/重命名操作完成后列表即时刷新
- 终端输出通过 PTY WebSocket 实时传输
- 网络断线时显示重连提示，恢复后自动同步状态

## 非功能需求

### NFR-1: 性能

- 终端输入到输出延迟不超过 100ms（本地网络）
- Session 列表加载时间不超过 500ms
- 分屏拖拽操作流畅，不低于 30fps
- 同时打开 5 个以上终端窗格不出现明显卡顿

### NFR-2: 可用性

- 所有操作有明确的视觉反馈（加载状态、成功/失败提示）
- 关键操作（删除）有确认步骤防止误操作
- 键盘快捷键支持（分屏、切换 Tab、切换面板焦点）
- 空状态和错误状态有友好的引导信息

### NFR-3: 兼容性

- 桌面端：Chrome、Firefox、Safari、Edge 最新两个主版本
- 移动端：iOS Safari 15+、Android Chrome 最新版本
- 最小视口宽度：320px（iPhone SE）
- 最大支持同时 128 个分屏窗格（实际受浏览器性能限制）

---

## 成功标准

1. **空状态引导完成率**: 新用户从看到空状态页到成功创建第一个 session 的操作步骤不超过 3 步
2. **Session 管理效率**: 创建、删除、重命名操作的端到端时间不超过 3 秒（从点击到 UI 反馈）
3. **终端交互体验**: 用户在浏览器终端中的输入响应延迟与本地终端差距不超过 50ms（本地网络）
4. **分屏操作流畅度**: 分屏拖拽调整大小时帧率不低于 30fps
5. **移动端可用性**: 手机用户可以完成列表查看、创建 session、进入终端、输入命令的完整流程
6. **多窗口并行**: 支持同时打开至少 5 个终端窗格且无明显性能下降
7. **布局适配覆盖**: 界面在 320px~2560px 宽度范围内均正常显示，无内容溢出或不可交互区域

## 关键实体

### TerminalInstance

表示一个浏览器内的终端实例（xterm.js 实例 + 对应的 WebSocket PTY 连接）。

| 属性 | 说明 |
|------|------|
| instanceId | 唯一标识（客户端生成的 UUID） |
| sessionName | 关联的 tmux session 名称 |
| windowIndex | 关联的 tmux window 索引 |
| paneId | 对应的 tmux pane ID（可选） |
| cols | 终端列数 |
| rows | 终端行数 |
| wsConnection | WebSocket 连接状态 |

### SplitLayout

表示分屏布局配置。

| 属性 | 说明 |
|------|------|
| id | 布局节点 ID |
| direction | 分屏方向（horizontal / vertical） |
| ratio | 分屏比例（0~1） |
| children | 子节点（递归结构，叶子节点为 TerminalInstance） |

### UserSettings

用户偏好设置（存储在 localStorage）。

| 属性 | 说明 |
|------|------|
| fontSize | 终端字体大小（12~24） |
| accentColor | 强调色（默认 #00ff41） |
| defaultSplitDirection | 默认分屏方向 |
| sidebarCollapsed | 侧边栏是否折叠 |

## 约束与假设

### 约束

- 必须使用项目现有的 React 19 + TypeScript + Tailwind CSS 4 技术栈
- 图标统一使用 Iconify（@iconify/react），不引入其他图标库
- WebSocket 基础设施复用 spec 1 中已定义的架构
- 后端 Rust/Axum 框架不变，新增的 PTY 端点遵循现有路由模式
- Logo 为纯 CSS/SVG 实现的像素字或艺术字，不使用外部图片资源

### 假设

- 用户设备已安装 tmux（否则进入 spec 1 定义的依赖安装引导流程）
- 单用户场景：不考虑多用户同时操作的权限冲突
- 本地或局域网使用：不优化高延迟广域网场景
- 浏览器支持 WebSocket 和现代 CSS 特性（Grid、Flexbox、CSS Variables）
- 终端字体（JetBrains Mono）通过 Web Font 加载，回退到系统等宽字体

## 范围边界

### 包含

- Session 的 CRUD 管理 UI
- xterm.js 终端集成与 PTY WebSocket 通信
- 分屏、多窗口 Tab、拖拽调整
- 移动端响应式适配
- 像素风格 Logo 设计
- 基础设置面板

### 不包含

- tmux 插件管理
- SSH 远程连接（仅管理本地 tmux）
- 用户认证/权限系统
- 终端录屏/回放功能
- 终端共享（多人协作同一终端）
- 自定义主题编辑器（仅提供预设强调色选择）
