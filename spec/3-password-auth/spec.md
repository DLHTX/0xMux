# 功能规格说明：密码鉴权与公网安全访问

**版本**: 2.0
**日期**: 2026-02-09
**状态**: 草案
**分支**: `3-password-auth`

---

## 概述

为 0xMux 添加密码鉴权系统，使其可以安全地暴露到公网。所有鉴权相关操作（设置密码、修改密码、管理外部访问）均通过 Web GUI 完成，不涉及任何命令行参数或环境变量。首次访问时通过弹框引导用户设置密码，后续通过登录弹框输入密码。同时将现有的设置页面（ThemeConfigurator）重构为带 Tab 的设置弹框，新增"安全"和"外部访问"Tab。

---

## 目标用户

- **主要用户**: 需要通过公网或局域网远程访问 0xMux 的开发者
- **次要用户**: 通过反向代理（Nginx/Caddy）或 VPN（Tailscale）暴露 0xMux 的运维人员

---

## 用户场景

### 场景 1：首次访问 — 设置密码弹框

**角色**: 开发者
**前置条件**: 首次启动 0xMux，未设置过密码
**流程**:

1. 用户启动 0xMux 服务，在浏览器打开界面
2. 页面加载后，弹出"设置密码"弹框（Modal），覆盖在主界面之上
3. 弹框标题："设置访问密码"，说明文案："为 0xMux 设置密码以保护你的终端访问"
4. 弹框包含：
   - 密码输入框（带显示/隐藏切换）
   - 确认密码输入框
   - 密码强度指示器（弱/中/强）
   - "设置密码"按钮
5. 两次输入不一致时，实时显示行内错误提示
6. 用户设置密码后，弹框关闭，自动完成登录，进入主界面
7. 密码通过 API 保存到服务端配置文件

**预期结果**: 首次访问通过 GUI 弹框引导设置密码，流程自然，无需任何命令行操作

### 场景 2：未登录访问 — 登录弹框

**角色**: 开发者
**前置条件**: 密码已设置，但用户未登录（无有效 token）
**流程**:

1. 用户在浏览器打开 0xMux
2. 页面加载后，弹出"登录"弹框（Modal），背景为模糊的主界面
3. 弹框包含密码输入框和"登录"按钮
4. 输入框支持 Enter 键提交
5. 密码正确：弹框关闭，进入主界面，token 存入 localStorage
6. 密码错误：输入框抖动 + 行内错误提示"密码错误"
7. 连续错误 5 次：提示"操作过于频繁，请 N 分钟后再试"，按钮禁用并显示倒计时

**预期结果**: 登录体验为弹框形式，轻量直接

### 场景 3：已登录用户的日常使用

**角色**: 已登录的开发者
**前置条件**: 用户已成功登录，浏览器中存有有效的 session token
**流程**:

1. 用户打开 0xMux，浏览器自动从 localStorage 读取 token
2. 所有 API 请求自动附带鉴权信息
3. 用户正常使用所有功能，无额外鉴权感知
4. 如果 token 过期或失效，自动弹出登录弹框

**预期结果**: 鉴权对已登录用户完全透明

### 场景 4：设置弹框 — 修改密码

**角色**: 已登录的开发者
**前置条件**: 用户已登录
**流程**:

1. 用户点击 Header 右上角的设置图标（齿轮）
2. 弹出设置弹框（Modal），包含多个 Tab：「外观」「安全」「外部访问」
3. 用户切换到「安全」Tab
4. 页面显示：
   - "修改密码"区域：当前密码输入框、新密码输入框、确认新密码输入框、"保存"按钮
   - "退出登录"按钮
5. 修改成功后显示成功提示，当前设备的 token 保持有效
6. 点击"退出登录"清除本地 token，弹出登录弹框

**预期结果**: 密码管理在设置弹框的安全 Tab 中完成

### 场景 5：设置弹框 — 外部访问管理

**角色**: 开发者
**前置条件**: 用户已登录
**流程**:

1. 用户打开设置弹框，切换到「外部访问」Tab
2. 页面显示：
   - "外部访问"开关（默认关闭）
   - 开关说明："允许从外部网络访问 0xMux。开启后服务将绑定到所有网络接口。"
   - 开启时显示安全提醒："请确保已设置强密码，建议配合 HTTPS 使用"
   - 当前监听地址显示（如 `127.0.0.1:1234` 或 `0.0.0.0:1234`）
   - 开启外部访问后显示：
     - 本机地址
     - 局域网 IP 地址（自动检测）
   - "危险操作"区域（仅外部访问开启时显示）：
     - "允许远程安装"开关（默认关闭）
     - "允许远程重启"开关（默认关闭）
3. 用户开启外部访问后，服务自动切换绑定地址（需重启服务生效）
4. 弹框提示"设置已保存，需要重启服务生效"，显示"立即重启"按钮

**预期结果**: 外部访问管理集中在专属 Tab 中，风险操作有明确的开关和警告

### 场景 6：服务重启后的鉴权连续性

**角色**: 开发者
**前置条件**: 用户已登录，服务发生重启
**流程**:

1. 服务因设置变更或手动重启而停止
2. 服务重启后，密码不变，已签发的 token 继续有效
3. 用户的浏览器自动重连，使用 localStorage 中的 token 重新鉴权
4. 如果密码在此期间被更改，旧 token 失效，弹出登录弹框

**预期结果**: 正常重启不中断用户的登录状态

## 功能需求

### FR-1：密码管理（纯 GUI）

密码通过 Web GUI 进行设置和管理，不涉及任何命令行参数或环境变量：

**首次设置**：
- 服务检测到未设置密码时，前端弹出"设置密码"弹框
- 弹框为强制性的，不可关闭（必须设置密码后才能使用）
- 密码要求：最少 6 个字符
- 包含密码强度指示器
- 需要二次确认输入

**存储**：
- 密码通过 `POST /api/auth/setup` 接口提交到服务端
- 服务端将密码哈希后存入 `~/.config/0xmux/config.toml`
- 配置文件权限设置为 `0600`（仅所有者可读写）

**修改**：
- 在设置弹框「安全」Tab 中修改
- 需要输入当前密码验证身份
- 修改成功后当前设备 token 保持有效，其他设备需重新登录

**验收标准**:

- 首次访问弹出设置密码弹框，不可跳过
- 密码不一致时显示行内错误
- 设置密码后自动登录
- 修改密码需验证当前密码
- 配置文件权限为 0600

### FR-2：登录认证接口

提供登录认证端点：

- **设置密码**: `POST /api/auth/setup` — 首次设置密码
  - 请求体: `{ "password": "...", "confirm": "..." }`
  - 已有密码时返回 409 Conflict
- **登录**: `POST /api/auth/login` — 验证密码
  - 请求体: `{ "password": "..." }`
  - 成功: `{ "token": "..." }`（HTTP 200）
  - 失败: `{ "error": "密码错误" }`（HTTP 401）
- **修改密码**: `PUT /api/auth/password` — 修改密码（需鉴权）
  - 请求体: `{ "current": "...", "password": "...", "confirm": "..." }`
  - 当前密码错误返回 401
- **鉴权状态**: `GET /api/auth/status` — 查询当前鉴权状态
  - 返回: `{ "initialized": true/false, "authenticated": true/false }`
  - 无需鉴权（前端据此决定显示设置密码弹框还是登录弹框）

Token 特性：
- 密码学安全的随机 64 字符 hex 字符串
- 有效期 7 天
- 支持多设备同时登录（多个有效 token）
- 服务重启后 token 继续有效（HMAC 签名，密钥从密码派生）

**验收标准**:

- 未初始化时 setup 接口可用，login 返回 403
- 初始化后 setup 返回 409，login 可用
- 正确密码返回 token，错误密码返回 401
- 修改密码需当前密码验证

### FR-3：请求鉴权中间件

所有 API 端点（除白名单外）均需通过鉴权中间件验证身份：

**鉴权方式**（按优先级检查）：
1. HTTP 请求头: `Authorization: Bearer <token>`
2. URL 查询参数: `?token=<token>`
3. Cookie: `mux_token=<token>`

**白名单端点**（无需鉴权）：
- `GET /api/health` — 健康检查
- `GET /api/auth/status` — 鉴权状态查询
- `POST /api/auth/setup` — 首次设置密码
- `POST /api/auth/login` — 登录
- 静态文件服务（前端 HTML/CSS/JS）

**鉴权失败响应**：
- 未提供 token: HTTP 401 `{ "error": "未登录" }`
- Token 无效或过期: HTTP 401 `{ "error": "登录已过期，请重新登录" }`

**验收标准**:

- 无 token 请求 API 返回 401
- 有效 token 通过三种方式均可成功鉴权
- 白名单端点无 token 也可访问
- 过期 token 返回 401

### FR-4：WebSocket 鉴权

WebSocket 连接在升级（upgrade）前完成鉴权验证：

- 三个 WebSocket 端点均需鉴权：`/ws`、`/ws/pty`、`/ws/install/{task_id}`
- 鉴权通过 URL 查询参数 `?token=<token>` 或 Cookie 完成
- 鉴权失败时拒绝 WebSocket 升级，返回 HTTP 401
- 连接建立后不再验证 token（连接期间 token 过期不断开已有连接）

**验收标准**:

- 无 token 的 WebSocket 连接被拒绝
- 有效 token 的 WebSocket 连接正常建立
- PTY WebSocket 鉴权后可正常收发终端数据

### FR-5：登录速率限制

防止暴力破解密码：

- 同一 IP 地址在 1 分钟内最多尝试登录 5 次
- 累计失败 5 次后，该 IP 被锁定 15 分钟
- 锁定期间的登录请求返回 HTTP 429 `{ "error": "操作过于频繁", "retry_after": N }`
- 成功登录不重置失败计数器
- 锁定到期后自动解除

**验收标准**:

- 1 分钟内第 6 次登录请求被拒绝（返回 429）
- 5 次失败后触发 15 分钟锁定
- 锁定期间即使密码正确也返回 429
- 锁定到期后可正常登录

### FR-6：CORS 安全策略

收紧跨源资源共享策略：

- 默认只允许与服务自身同源的请求
- 不再使用 `allow_origin(Any)`
- 开发模式下额外允许 Vite 开发服务器源（`http://localhost:3000`）

**验收标准**:

- 来自其他域的 API 请求被 CORS 拦截
- 自身源的请求正常通过
- 开发模式下前端开发服务器正常通过

### FR-7：外部访问管理（GUI）

在设置弹框「外部访问」Tab 中管理公网暴露策略：

**外部访问开关**：
- 默认关闭（服务绑定 `127.0.0.1`）
- 开启后服务绑定 `0.0.0.0`（所有网络接口）
- 切换后需重启服务生效，弹框内显示"立即重启"按钮
- 开启时显示安全提醒

**信息展示**：
- 当前监听地址
- 局域网 IP 地址（开启外部访问后自动检测并显示）

**危险操作开关**（仅外部访问开启时显示）：
- "允许远程安装"开关（默认关闭）— 控制 `POST /api/system/install`
- "允许远程重启"开关（默认关闭）— 控制 `POST /api/system/restart`
- 关闭时对应端点返回 403

**路径访问限制**：
- 外部访问模式下 `GET /api/dirs` 限制在 `$HOME` 范围内

**配置持久化**：
- 所有开关状态通过 API 存入服务端配置文件
- 服务重启后按配置文件的状态运行

**验收标准**:

- 外部访问开关切换后提示需重启
- 重启后绑定地址正确切换
- 危险操作开关关闭时对应端点返回 403
- 外部访问模式下 dirs 端点限制在 $HOME 内
- 局域网 IP 自动检测并显示

### FR-8：设置弹框重构

将现有的设置页面（ThemeConfigurator 侧边面板）重构为带 Tab 的设置弹框：

**弹框结构**：
- 居中的 Modal 弹框，带背景模糊遮罩
- 顶部标题栏："设置"，右侧关闭按钮
- Tab 栏：「外观」「安全」「外部访问」
- 内容区域根据当前 Tab 切换

**「外观」Tab**：
- 迁移现有 ThemeConfigurator 的全部内容（主题预设、颜色、字体、圆角等）
- 保持现有功能不变

**「安全」Tab**：
- 修改密码表单（当前密码 + 新密码 + 确认新密码）
- 退出登录按钮

**「外部访问」Tab**：
- 外部访问开关及相关设置（见 FR-7）

**验收标准**:

- 设置图标点击后弹出 Tab 式 Modal
- 外观 Tab 包含原 ThemeConfigurator 全部功能
- 安全 Tab 包含密码修改和退出登录
- 外部访问 Tab 包含外部访问管理
- Tab 切换流畅，状态保持

### FR-9：前端鉴权流程

前端根据鉴权状态自动显示对应的弹框：

**启动流程**：
1. 页面加载时调用 `GET /api/auth/status`
2. 如果 `initialized === false`：弹出"设置密码"弹框（不可关闭）
3. 如果 `initialized === true` 且本地无 token 或 token 无效：弹出"登录"弹框（不可关闭）
4. 如果 token 有效：直接进入主界面

**Token 管理**：
- 存储在 localStorage，key 为 `mux_token`
- 所有 API 请求通过 `Authorization: Bearer <token>` 携带
- WebSocket 连接通过 `?token=<token>` 参数携带
- API 返回 401 时自动清除 token 并弹出登录弹框

**验收标准**:

- 首次访问显示设置密码弹框
- 未登录显示登录弹框
- 已登录直接进入主界面
- token 过期自动弹出登录弹框
- 弹框为强制性，不可通过点击背景关闭

## 非功能需求

### NFR-1：安全性

- 密码在服务端哈希存储（不存明文）
- Token 使用密码学安全的随机数生成器
- 配置文件权限 0600
- 速率限制状态存储在内存中（服务重启后清零）
- 不记录密码和 token 到日志文件
- 公网部署建议配合 HTTPS 反向代理

### NFR-2：性能

- 鉴权中间件对请求延迟的影响不超过 1ms
- 速率限制检查不超过 0.1ms
- Token 验证为 O(1) 操作

### NFR-3：兼容性

- 与现有所有 API 端点和 WebSocket 连接兼容
- 不改变现有端点的请求/响应格式（仅增加鉴权层）
- 设置弹框重构后保留原有外观设置的全部功能

## 成功标准

1. 首次访问到完成密码设置不超过 30 秒（弹框引导，零命令行操作）
2. 已登录用户的使用体验与无鉴权时完全一致
3. 暴力破解在速率限制下不可行
4. 通过公网暴露后，未登录用户无法执行任何操作
5. 所有安全设置均可在 GUI 中完成，无需编辑配置文件或使用命令行参数

## 关键实体

| 实体         | 说明                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 密码         | 服务的访问密码，通过 GUI 设置，哈希后存储在服务端配置文件             |
| Session Token| 登录成功后签发的访问令牌，64 字符 hex，有效期 7 天                   |
| 配置文件     | `~/.config/0xmux/config.toml`，存储密码哈希和服务配置                |
| 速率限制记录 | 按 IP 地址记录的登录尝试次数和锁定状态，存储在内存中                 |
| 外部访问配置 | 绑定地址、危险操作开关等，存储在服务端配置文件                       |

## 假设

1. 0xMux 是单用户工具，只有一个密码，不需要多用户账号
2. HTTPS 由反向代理负责，0xMux 本身不内置 TLS
3. 配置文件使用 TOML 格式
4. 速率限制数据存储在内存中，服务重启后清零
5. Token 有效期 7 天
6. 首次访问必须设置密码，不提供"跳过"选项
7. 外部访问开关切换后需要重启服务才能生效（绑定地址是启动时确定的）
