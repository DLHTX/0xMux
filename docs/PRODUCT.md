# 0xMux — 产品设计文档

## 一句话定义

面向开发者的 tmux Web 管理面板，用浏览器替代终端命令来管理 session，PC 和手机通用。

---

## 痛点

### 1. tmux 管理混乱

桌面端使用 tmux 管理多个终端会话，体验极差：

- 创建 session 后经常忘记关闭，**僵尸 session 长期驻留、占用内存**
- 查看当前有哪些 session 需要手动 `tmux ls`，没有可视化概览
- 切换/关闭/重命名都靠命令行，操作繁琐且容易误操作
- 多个 window/pane 嵌套后，**层级关系完全靠记忆**

### 2. Claude Code 使用不便

- 需要安装大量插件才能正常使用
- 显示技能列表、查看状态等操作都依赖命令输入
- 多个 Claude Code 会话并行时，窗口切换混乱
- 没有一个统一的面板来管理所有 AI 会话

### 3. 没有好用的现有工具

- 市面上的 tmux Web UI 项目要么太老、要么 star 太少、要么根本没人维护
- 少数可用的（如 Agentboard）功能单一，设计风格一般
- **这个领域存在明显空白**

---

## 产品定位

```
目标用户：使用 tmux + Claude Code 的开发者
核心场景：在浏览器中管理所有终端会话
设计风格：极客 / 暗色 / 霓虹绿
平台支持：PC 浏览器 + 手机浏览器（响应式 + PWA）
```

---

## 功能规划

### Phase 1：基础 Session 管理

> 让 tmux 管理从命令行搬到浏览器

| 功能 | 说明 |
|------|------|
| Session 列表 | 实时展示所有 tmux session，显示名称、窗口数、状态（attached/detached） |
| 创建 Session | 一键创建新 session，支持自定义命名 |
| 关闭 Session | 单个关闭 / 批量关闭 |
| 重命名 Session | 点击编辑 |
| 状态指示 | 绿色 = attached，灰色 = detached，实时刷新 |
| 响应式布局 | PC 三列卡片网格，手机单列堆叠 |

### Phase 2：内嵌终端 + 智能管理

> 不只是看，还能直接操作

| 功能 | 说明 |
|------|------|
| 内嵌终端 | 基于 xterm.js，点击 session 卡片直接打开终端，通过 WebSocket 实时交互 |
| 僵尸检测 | 自动识别长时间无活动的 session，提醒清理或自动回收 |
| Session 分组 & 标签 | 给 session 打标签（如 `work` `ai` `debug`），按组筛选 |
| Window/Pane 可视化 | 展示 session 内部的 window 和 pane 布局结构 |

### Phase 3：开发者工具集成

> 从 tmux 管理工具进化为开发者工作台

| 功能 | 说明 |
|------|------|
| 目录浏览器 | 显示用户工作目录结构，快速定位项目 |
| Claude Code 识别 | 自动检测哪些 session 在运行 Claude Code |
| 技能展示 | 显示当前 Claude Code 可用的技能列表 |
| AI 会话概览 | 聚合所有 AI agent 的状态，统一管理 |

### Phase 4：协作 & 远程

> 走出本机

| 功能 | 说明 |
|------|------|
| PWA 支持 | 手机添加到主屏幕，像原生 App 一样使用 |
| 远程访问 | 通过局域网 / Tailscale 从手机管理 PC 上的 tmux |
| 多机管理 | 连接多台机器的 tmux，统一面板 |

## 技术栈

```
┌─────────────────────────────────────────────────┐
│                    0xMux                         │
├────────────────────┬────────────────────────────┤
│    Frontend        │    Backend                  │
│                    │                             │
│    React 19        │    Rust                     │
│    TypeScript      │    Axum (HTTP + WebSocket)  │
│    Tailwind CSS    │    Tokio (异步运行时)        │
│    xterm.js        │    Serde (序列化)            │
│    Shadcn/UI       │                             │
├────────────────────┼────────────────────────────┤
│    通信             │                             │
│                    │                             │
│    REST API ──────→ Session CRUD / 配置          │
│    WebSocket ────→ 终端实时数据流                 │
├────────────────────┴────────────────────────────┤
│    部署                                          │
│                                                  │
│    Bun (前端构建)                                 │
│    Cargo (后端构建)                               │
│    单二进制分发 (Rust 内嵌前端静态文件)             │
└─────────────────────────────────────────────────┘
```

### 为什么选 Rust

- **性能**：终端数据流吞吐量大，Rust + Tokio 的异步性能远超 Node/Python
- **单二进制**：编译后一个文件搞定，不需要运行时，分发简单
- **安全**：内存安全，适合长时间运行的服务端程序

### 为什么选 React + Tailwind

- **生态**：xterm.js 原生支持 React，Shadcn/UI 组件质量高
- **Tailwind**：暗色极客风格定制方便，响应式开箱即用
- **PWA**：Vite 生态 PWA 插件成熟，手机端体验好

## 设计风格

```
主题：暗色极客 (Dark Hacker)
主色：#0a0a0a (纯黑背景)
强调色：#00ff41 (霓虹绿，Matrix 风格)
字体：JetBrains Mono (等宽，代码感)
边框：细线 + 微弱霓虹光晕
动效：最小化，仅状态变化时有呼吸灯效果
```

面向开发者，不需要花哨装饰。信息密度高，操作路径短。

## 差异化

| 对比 | Agentboard | Webmux | **0xMux** |
|------|-----------|--------|-----------|
| 后端 | Node.js | Rust | **Rust** |
| 前端 | React | Vue | **React** |
| 设计 | 普通 | 普通 | **极客霓虹风** |
| 手机支持 | 有 | PWA | **PWA + 响应式** |
| Session 管理 | 基础 | 基础 | **智能（僵尸检测、分组、标签）** |
| AI 集成 | Claude 识别 | 无 | **技能展示 + 多 AI 管理** |
| 目录浏览 | 无 | 无 | **有** |
| 分发方式 | npx | 编译 | **单二进制** |
